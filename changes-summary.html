NEW foodseq_reference_pipeline.Rmd
  • Replaces separate trnL-reference.Rmd and 12SV5-reference.Rmd with a single document containing unified setup, shared SLURM jobs, and parallel Part A (trnL) / Part B (12SV5) sections
  • SLURM job submission writes scripts with writeLines() and prints the sbatch command for manual execution — avoids PATH errors when system("sbatch") is called from an OOD RStudio session
  • Animal species list read dynamically from human-foods.csv rather than hardcoded in the script
  • Auto-resolve missing taxonomy IDs: loop over accessions with NA taxon IDs and call query_ncbi_accession() to retrieve them from NCBI, covering accessions added after the SQL database was built
  • NCBI API key chunk added; sources all three function files; shared extract_taxonomy() helper used by both Part A and Part B
  • Manual curation edits from Manual renaming.csv integrated into pipeline at A5 (manual additions) and A7 (omissions and renamings)
MODIFIED code/functions/query_ncbi.R
  • Sequence-length filter 0:50000[SLEN] added to NCBI search query Root cause of crashes: chromosomal/scaffold assemblies containing the 12S gene were included in results; these can be megabytes each and push the FASTA payload over R's 231−1 byte string limit. The filter excludes them while retaining all genuine amplicons and gene-level sequences.
  • Retry logic (fetch_with_retry()) wraps every entrez_fetch() call — retries up to 4 times with a 10 s wait on SSL / network drops instead of aborting the whole query
  • retmax added to entrez_fetch() — caps sequences downloaded per chunk as a secondary size guard
  • Chunk size reduced 50 → 5 species per query to reduce per-fetch payload size
  • 0.5 s pause between chunks to respect NCBI rate limits; progress messages updated to show chunk N of total
  • Key parameters (chunk_size, max_seq_len, retmax_fetch, max_tries, retry_wait) exposed as function arguments for easy tuning
MODIFIED code/functions/find_primer_pair.R
  • Added relationship = "many-to-many" to both inner_join() calls Newer versions of dplyr warn when a many-to-many join is detected without this argument. Finding multiple primer binding sites per genome sequence is expected behaviour; the argument makes this explicit and silences the warnings. No change to results.
NEW code/functions/query_ncbi_accession.R
  • Added to code/functions/; called in the taxonomy lookup steps (A6, B6) to resolve accession numbers that are absent from the local taxonomizr SQL database because they were added to NCBI after the database was built

NEW data/outputs/ — May 2026 database run
  • Pipeline run on Duke HPC cluster (Open OnDemand / SLURM) using the updated foodseq_reference_pipeline.Rmd. Output files labelled with _May2026 suffix to record when the databases were last compiled.
  • trnL (plants): 1,991 sequences · 1,169 unique taxa · 1,060 / 1,577 food plant species covered (67%) Unchanged from the 2025 run — RefSeq plastid and GenBank trnL content for these species is stable.
  • 12SV5 (vertebrates): 3,390 sequences · 2,337 unique taxa · 1,168 / 2,095 food animal species covered (56%) +399 sequences and +103 newly covered food species versus the 2025 run, reflecting both new GenBank deposits and improvements to the query and taxonomy-lookup logic.
  • Coverage gap lists added: plants_missing_trnL.csv and animals_missing_12SV5.csv document the 517 plant and 927 animal species still lacking sequences in the current databases
